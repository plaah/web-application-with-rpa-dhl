## Interaction Contract

### Expand/Collapse behavior
- Each run group is collapsed by default on page load
- Clicking the group header row toggles that group's expanded state
- State managed with `useState<Set<string>>` — set of expanded `run_id` values
- No "expand all / collapse all" toggle (deferred to Claude's Discretion — individual toggle is simpler and sufficient)
- ChevronDown shown when collapsed; ChevronUp shown when expanded

### Data fetching
- `useEffect` on mount: `apiFetch<LogsResponse>("/logs?limit=200")`
- Group client-side by `run_id` using `reduce`
- Sort groups by most recent `timestamp` in group descending
- Loading state: single centered `text-sm text-gray-500` spinner text "Loading logs..." (no spinner component — matches existing page patterns)
- No pagination controls in this phase (limit=200 covers demo scale)

### Empty state
- Rendered when `logs.length === 0` after fetch completes
- See Copywriting Contract below

### Error state
- On `apiFetch` throw: caught in try/catch, sets `error` state string
- Renders error message as per Copywriting Contract below
- `apiFetch` already handles 401 → redirect to /login (no extra handling needed)
