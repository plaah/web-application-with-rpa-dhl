Base URL: `http://localhost:8000/api` (read from NEXT_PUBLIC_API_URL fallback).
Auth: `Authorization: Bearer <jwt>` header on every call except `/api/auth/login` and `/api/health`.
Cookie name: `irrs_token` (set client-side after login, read by middleware.ts).

Endpoints:
- POST /api/auth/login — body `{username, password}` → `{success, data: {token, user: {id, username, role}}}`. Failure: 401.
- GET /api/incidents?status=&tag=&creator=&date_from=&date_to=&search=&page=1&limit=20 → `{success, data: {incidents: Incident[], total, page, limit}}`
- POST /api/incidents (multipart/form-data: title, description, priority, category, tags, source, raw_content, content_hash, files[]) → 201 `{success, data: {incident}}`
- GET /api/incidents/{id} → `{success, data: {incident}}`. 404 if missing.
- PUT /api/incidents/{id} (JSON, draft-only) → 200 / 403 if not draft
- DELETE /api/incidents/{id} (admin + draft only) → 200 / 403
- PATCH /api/incidents/{id}/status — body `{status: "reviewed"|"published"}` → 200 / 400 invalid transition
- POST /api/files/extract (multipart, field `file`) → `{success, data: {extracted_text, file_type}}`
- GET /api/reports/summary → `{success, data: {total_incidents, by_status, by_priority, by_category, rpa_runs, last_rpa_run}}`
