# AGENTS.md

This file provides guidance to Codex (Codex.ai/code) when working with code in this repository.

## Project

DHL Incident Reporting & Resolution System (IRRS) — university assignment. FastAPI backend (port 8000) + Next.js 16 frontend (port 3000). No real database; all persistence is JSON files with `filelock`.

## Dev Commands

**Backend** (always activate venv first):
```bash
cd irrs-backend
source venv/bin/activate
uvicorn main:app --reload --port 8000

# Reseed users (wipes users.json):
python seed.py
```

**Frontend:**
```bash
cd irrs-frontend
npm run dev        # port 3000
npm run build
npm run lint
```

**API docs** (auto-generated by FastAPI): http://localhost:8000/docs

## Architecture

### Backend (`irrs-backend/`)

FastAPI app in `main.py` mounts 6 routers under `/api`:

| Router | Prefix | Responsibility |
|--------|--------|----------------|
| `auth.py` | `/api/auth` | Login, returns JWT |
| `incidents.py` | `/api/incidents` | Full CRUD + status PATCH + dedup check |
| `files.py` | `/api/files` | Upload + text extraction (PDF/DOCX/TXT) |
| `reports.py` | `/api/reports` | Aggregated summary counts |
| `logs.py` | `/api/logs` | RPA execution logs |
| `ai.py` | `/api/ai` | AI suggestion stub (Phase 3) |

**Storage** — `utils/storage.py`: `read_json(filename)` / `write_json(filename, data)` — reads/writes `data/*.json` with `FileLock`. All reads return a Python list; callers mutate in memory then call `write_json`.

**Auth** — `utils/auth_utils.py`: `HTTPBearer(auto_error=False)` + explicit `HTTPException(401)` in `get_current_user`. The default FastAPI `auto_error=True` returns 403 on missing token — this was intentionally disabled. All protected routes use `Depends(get_current_user)`.

**JWT**: HS256, 8h expiry. `SECRET_KEY` read from env (defaults to `dhl-irrs-dev-secret-change-me`). Set a real key in `irrs-backend/.env`.

**Incident status machine** (`routers/incidents.py`):
```
draft → reviewed → published   (one-way only)
```
`VALID_TRANSITIONS` dict enforces this; backwards PATCH returns 400. Editing (`PUT`) is blocked when status != draft (returns 403). Every status change and edit appends to `version_history`.

**File parsing** — `utils/file_parser.py`: use `import fitz` (not `import pymupdf`) for PDFs — that's how PyMuPDF exposes itself in this version.

**Dedup** — `utils/dedup.py`: SHA-256 hash of content, 14-day window checked against `data/hashes.json`.

**All API responses** use this envelope:
```json
{ "success": true, "data": ..., "message": "..." }
```

**Seeded users** (via `seed.py`):
| username | password | role |
|----------|----------|------|
| admin | admin123 | admin |
| editor | editor123 | editor |
| rpa_bot | rpabot123 | rpa_bot |

**Roles**: `admin` — full access including DELETE; `editor` — create/edit drafts; `rpa_bot` — automated ingestion via RPA (Phase 2).

### Frontend (`irrs-frontend/`)

Next.js 16 App Router. No component library — plain Tailwind 4 + `lucide-react`.

**Auth flow**: `middleware.ts` guards all routes except `/login`. On 401 from any `apiFetch` call, cookies are cleared and the browser is redirected to `/login`. Auth state lives in two cookies:
- `irrs_token` — JWT, 8h, used as `Authorization: Bearer` header
- `irrs_user` — JSON-encoded `{id, username, role}`, drives role-based UI

**API client** — `lib/api.ts`: `apiFetch<T>(path, options)` wraps `fetch`, injects the bearer token from `irrs_token` cookie, and skips `Content-Type: application/json` for `FormData` bodies. Base URL: `NEXT_PUBLIC_API_URL` env var (defaults to `http://localhost:8000/api`).

**Page → route map**:
| File | Route |
|------|-------|
| `app/page.tsx` | `/` → redirects to `/dashboard` |
| `app/login/page.tsx` | `/login` |
| `app/dashboard/page.tsx` | `/dashboard` |
| `app/upload/page.tsx` | `/upload` |
| `app/incidents/page.tsx` | `/incidents` |
| `app/incidents/[id]/page.tsx` | `/incidents/:id` |

**Incident detail page** (`app/incidents/[id]/page.tsx`) calls `fetchIncident()` after every status PATCH to update the UI in-place without a page reload.

**CORS**: backend allows only `http://localhost:3000`. Don't add other origins without updating `main.py`.

## Key Invariants

- Never set `auto_error=True` on `HTTPBearer` — it breaks the 401 contract.
- `import fitz` for PDF, not `import pymupdf`.
- JSON data files are in `irrs-backend/data/` — always use `read_json`/`write_json`, never open them directly (no locking).
- Status transitions are one-way; don't add backwards paths without updating the frontend state machine too.
